# Настройка TLS для веб-интерфейса Proxmox VE 9

Настроем получение доверенного TLS-сертификата для веб-интерфейса Proxmox VE 9 с помощью Let's Encrypt.   
Нам понадобится доменное имя (осторожнее с выбором, после покупки `kropachev.digital` мне пришлось начать вести блог).  
Регистратором я выбрал REG.RU, потому что у них есть api, который можно использовать для автоматического выпуска сертификатов.  
В качестве способа подтверждения владения доменом используется DNS-01 challenge.

⚠️ Несмотря на доменное имя, в моем случае `pve.kropachev.digital`, доступ будет только внутри домашней сети, я не открываю прямой доступ из интернета.

## Предпосылки

Инструкция рассчитана на сценарий, когда Proxmox установлен на домашнем сервере и доступен только из локальной сети.

Исходные данные:

* установленный Proxmox VE 9;
* доменное имя (в примере`kropachev.digital`);
* DNS-зона домена обслуживается у REG.RU (из-за наличия API);
* есть доступ к REG.RU API (логин и пароль для API);
* Proxmox имеет доступ в интернет.

## Выбор доменного имени для Proxmox

Для удобства работы и выпуска сертификата нужно выделить отдельное доменное имя для веб-интерфейса Proxmox.

Например:

* `pve.kropachev.digital`

Именно по этому имени мы будем попадать в веб-интерфейс Proxmox и именно для этого доменного имени будет выпущен сертификат:

* в браузере нужно будет открывать адрес `https://pve.kropachev.digital:8006`;
* доступ по IP-адресу (`https://192.168.x.x:8006`) работать будет, но сертификат не будет считаться валидным для IP.

Т.к. доступ предполагается только внутри сети, то DNS мы настраиваем в роутере.

В OpenWRT это делается в настройках **Network** - **DHCP and DNS**  
В поле Addresses нужно указать соответствие Адреса и IP.  
Пример для IPv4 и IPv6:
```
/pve.kropachev.digital/192.168.1.50
/pve.kropachev.digital/::ffff:192.168.1.50
```
Чтобы эти настройки применились нужно перезапустить роутер.

## Подготовка учетных данных REG.RU API

Для автоматического создания TXT-записей в DNS понадобятся учетные данные REG.RU API. Они передаются в ACME-плагин в виде переменных окружения. Кроме того, в апи нужно выполнить базовые настройки.

В личном кабинете reg.ru для перехода к настройкам нужно кликнуть по названию своего аккаунта вверху-справа и выбрать `Настройки`.  

В списке выбираем раздел `Настройки API`.  

Во-первых, нам нужно указать `Альтернативный пароль`, пароль который будет использоваться для API, чтобы не светить пароль доступа к личному кабинету.

Во-вторых, необходимо задать `Диапазоны IP-адресов` - IP адреса, с которых будет разрешено использовать апи. Есть возможность кликнуть `Мой IP`. Только с этих адресов API будет доступен.

Теперь API сможет обрабатывать наши запросы.

## ACME-аккаунт Let's Encrypt

ACME-аккаунт используется для взаимодействия с Let's Encrypt.  
Переходим в веб-интерфейс Proxmox.

В левой панели выбираем **Datacenter** и раздел **ACME**.

Вверху, в подрезделе **Accounts** жмем Add и заполняем поля:
1. **Account Name** - любое имя (regru);
2. **E-Mail** - свой имейл;
3. **ACME Directory** - Let's Encrypt V2;
4. **Accept TOS** - ✅.

Жмем Register.

## DNS-плагин REG.RU

Следующий шаг - создание DNS-плагина, который будет использовать REG.RU API для создания и удаления TXT-записей.

Здесь же, в разделе **ACME** расположен раздел **Challenge Plugins**.

Жмем Add

1. **Plugin ID** - произвольное имя, например `regru-dns`.
2. **Validation Delay** - я ставлю 600 секунд (ну очень долго подтверждение проходит).
3. **DNS API** - в списке выбираем regru.
4. **API Data** - вставляем значения переменных:
   ```
   REGRU_API_Username=your_login
   REGRU_API_Password=your_api_password
   ```
Все. Жмем OK.

## Заказываем сертификат

Сертификат заказывается отдельно для каждого узла.

Выбираем нужный узел (скорее всего он один), и переходим в **System** - **Certificates**.  
В разделе **ACME** жмем Add и заполняем поля для домена:
1. **Challenge Type** - DNS;
2. **Plugin** - выбираем созданный ранее плагин `regru-dns`.
3. **Domain** - доменное имя, для которого выпускаем сертификат и по которому будем переходить в наш Proxmox, например`pve.kropachev.digital`;

Подверждаем Create и в списке появляется новая запись с нашим доменом.  
Теперь собственно сам заказ сертификата - жмем **Order Certificates Now**.
Немного ждем (600 секунд) и готово.

## Проверка работы сертификата

Открываем страницу с нашим доменом и портом - браузер не показывает ошибок сертификата. 🤷‍♂️

## Автоматическое продление сертификата

Let's Encrypt выдает сертификаты с ограниченным сроком действия. Proxmox автоматически следит за сроком действия ACME-сертификатов и продлевает их через системные задачи.
